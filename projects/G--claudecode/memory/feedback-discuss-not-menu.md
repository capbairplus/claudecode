決定下一步/方向時的**節奏**(使用者很在意這個):

1. **先跟他討論**——聊清楚他要什麼、給我的看法。
2. 談得差不多**再列選項**——數字條列 1.2.3.4 **完全 OK**,他明說可以丟。
3. **列完就停住、等他回**:不逼他當下決定、不跳「一定要選一個」的強制框、**也不要自己先把事情往下做(例如列完馬上去存記憶/執行)**。
4. 由**他**決定下一步:覺得不夠 → 再討論;其中一項正是他要的 → 他會直接說「選第幾個」。

5. **開不開生/開不開工由他說**:我**不催、不用「要不要開始/敲定就開生」來收尾**。他叫我開,我才開;他沒說,我就討論完停住等。動作的發起權在他,不是我。

**最容易犯的錯**:把我的理解或選項打出來後,**同一輪就自己動手(存記憶/開始做)**,或在句尾催他開工,等於逼著往下走。正解是**先問「這樣對嗎?要不要修?」然後真的停下來等他確認**,他點頭我才動作。

**分寸**:整體要有分寸——別急、別推、別自作主張、別在敏感或未定案的事上搶著往下衝;寧可少做、停下等他。

搭配 [[feedback-only-do-what-asked]](不自作主張)、[[feedback-batch-gen-workflow]](批次生圖一次派遣一次收圖):少打擾、討論式、他來主導節奏。
