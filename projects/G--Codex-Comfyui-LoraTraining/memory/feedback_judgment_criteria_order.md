被使用者問「一般人/使用者會怎麼判斷要不要採用某個工具、skill、套件」這類問題時,回答順序要先講最直覺、一般人實際會用的標準——star 數、下載/使用人數、其他使用者的評價——再視情況延伸到技術性/進階考量(程式碼審查、資料外流風險、License、依賴鏈等)。

**Why:** 在討論「使用者要如何判斷是否安裝某個 GitHub 上的 token 優化 skill」時,第一次回答直接跳到程式碼可讀性、資料流向、執行權限這類技術審查層級的考量,完全略過一般人第一直覺會看的 star 數/下載量/評價,被使用者指出這是「脫離正常人判斷的最基本標準」。使用者本身也曾表明沒有能力自己審查程式碼(見情境:他說「我沒那能力去做 skill」),代表技術性判斷依據對他來說優先度應該更低,不該擺在前面。

**How to apply:** 任何「怎麼判斷/評估要不要採用某工具」的問題,先給:1) star/下載量等社會認同訊號 2) 使用者評價/踩雷回報,再補充技術性考量。不要預設使用者有能力自行審查程式碼或評估資安風險,除非對話中已確認他有這個能力。
