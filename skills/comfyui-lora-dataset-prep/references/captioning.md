# 打標 / Caption 策略

## 核心心智模型

caption 是在告訴模型:**畫面裡哪些是「變數」、哪些是要學進觸發詞的「常數」。**

- **想學進觸發詞的特徵 → 不要打出來**(打出來 = 告訴模型「這是可變的、由這個詞控制」,
  反而學不進觸發詞)。例:練某個人的臉,就別細打他的五官。
- **會變動、不想綁死的 → 打出來**(背景、服裝、姿勢、光線),讓模型知道這些與觸發詞無關,
  生圖時才能自由改。

## 依類型的策略

### 人物 / 臉
- 每張:`<觸發詞>, <會變動的東西>`,例如 `mytrigger, smiling, side view, outdoor background`。
- 臉部特徵少打,讓它綁進觸發詞;背景/服裝/表情/角度打出來以便分離。

### 風格
- **短 caption + 專屬罕見觸發詞**。描述「畫了什麼內容」(a cat, a mountain),
  **不要**描述風格本身(不要寫 "watercolor style")——風格要靠觸發詞學,寫出來反而稀釋。
- 素材題材要多樣,否則會把「常出現的內容」一起學成風格的一部分。

### 概念 / 多概念(重要實戰教訓)
一個 LoRA 想同時分多個概念(例:15 個朝代甲冑)時:
- **每個概念一個「專屬、罕見」的觸發詞**(如 `cdwqin` / `cdwtang` / `cdwming`),
  不要用通用詞(`qin dynasty`)——通用詞會撞底模先驗,概念間互相污染。
- **caption 極簡**:`<trigger>, ancient chinese armor` 這種程度就好。
- **分概念放各自資料夾**,用 repeats 平衡各概念張數。
- 教訓來源:曾用「通用詞 + Florence 長自然語言 caption」訓 Flux,結果 15 個概念
  長得都差不多、彼此不分。長句 caption + 通用觸發詞會把各概念「平均」成一團。
  → **分概念風格/多概念 LoRA 的 caption 要「短 + 專屬罕見觸發詞」。**

## 打標工具

- **WD14 tagger**(`SmilingWolf/wd-vit-tagger-v3` 等):animescore/插畫最合適。
  - 注意:對**非動漫主體**會產生雜訊標(japanese_clothes、blue_hair 等)。
    v1 可先容忍,迭代時再建黑名單過濾。
  - 常見 bug:標籤清單前面若混入 rating 假項目會導致整體位移錯位——檢查對齊。
- **Florence-2 / BLIP**:產自然語言描述。適合寫實單概念;**風格/多概念慎用**(見上)。
- 一律**每句前面補觸發詞**。

## 殘留文字 / 浮水印 / 引線標註
掃描書頁或素材常有文字、箭頭、引線。兩條路:
1. **打標容忍**:caption 加 `text`,生圖時用負面詞壓制。省事,v1 常用。
2. **清圖**:IOPaint / inpaint 清除後再訓練。乾淨但費工。
先容忍、訓練測試後再決定要不要回頭清。

## 檔案規則
- caption 存成與圖**同檔名的 `.txt`**(`001.png` ↔ `001.txt`),一圖一檔,同資料夾。
- 編碼 UTF-8;跑打標腳本時 cp950 系統設 `PYTHONUTF8=1`。
